# Plan: auto-add a ground plane after routing (`--ground-plane`)

Status: **✅ Implemented.** All three phases shipped. Several bugs subsequently
found and fixed (thermal bridge width, connectivity via conflict detection,
THT pad connectivity). See `CHANGES.md` for full history.

## Goal

An opt-in step that, **after routing**, adds a **ground-net copper pour** that
follows the board outline (inset by a margin), and makes sure the ground net is
actually tied to it — adding connecting **vias** (or short tracks) where a GND
island doesn't already reach the pour layer. Optional **stitching vias** tie the
plane(s) together. If the board **already has a copper pour**, emit a **warning and
skip** (don't double-pour).

```
pyautoroute board.kicad_pcb --ground-plane \
    [--ground-net GND] [--ground-plane-layer back|front|both] \
    [--ground-plane-margin 0.5] [--stitch-vias [PITCH]]
```

## Motivation

A ground pour is near-universal on real boards (return paths, EMI, thermals) and is
tedious to add by hand after an auto-route. PyAutoRoute already routes everything,
already knows the outline and the netlist, and already shells out to `kicad-cli` to
fill zones — so it is well placed to drop in a correct GND plane and guarantee the
ground net connects to it, in one pass.

## Key insight: we emit the *boundary*, KiCad computes the *fill*

A KiCad `(zone …)` node carries a **user outline** `(polygon (pts …))` plus fill
settings; the actual filled copper `(filled_polygon …)` — which respects clearance
to other-net copper, thermal reliefs, island removal — is **computed by KiCad**.
PyAutoRoute already runs that fill via `pcb.try_refill_zones`
(`kicad-cli pcb drc --refill-zones --save-board`, pcb.py:763). So this feature only
needs to:

1. synthesize the **zone boundary node** (outline inset by a margin), and
2. add the **vias** that guarantee ground connectivity (PyAutoRoute's own domain),

then let the existing refill path fill it. We do **not** compute the pour copper
geometry ourselves.

> **Important limitation (call out clearly).** `geometry.clearance_violations` →
> `board_obstacles` **skips `fill_enabled` zones** (geometry.py:254: "copper pours
> are managed by KiCad"). So PyAutoRoute's in-repo DRC self-check **cannot verify
> the pour's clearance** to other-net copper — that is delegated to KiCad's fill /
> `kicad-cli` DRC. PyAutoRoute *can* still verify the connection/stitching **vias**
> (they go through the grid's clearance model). When `kicad-cli` is unavailable the
> zone is *defined but unfilled* — warn the user to open the board in KiCad to fill
> (same as today's `try_refill_zones` fallback).

## Reference: a real KiCad GND zone (TestProjects/Test1, ~line 6580)

```lisp
(zone
    (net "GND") (layer "B.Cu") (uuid "…")
    (hatch edge 0.5)
    (connect_pads (clearance 0.5))
    (min_thickness 0.25)
    (fill yes (thermal_gap 0.5) (thermal_bridge_width 0.5) (island_removal_mode 0))
    (polygon (pts (xy 39 37) (xy 77.5 37) (xy 77.5 81.5) … ))
    (filled_polygon (layer "B.Cu") (pts … ))   ; ← generated by KiCad on refill
)
```

We emit everything **except** `filled_polygon` (KiCad adds it). `(net …)` uses the
board's reference style via `_net_ref_node`; the `net_name`/`net` numbering must
match the board (numbered vs name-only). Clearances/thermals come from the rules.

## Components

### 1. Ground-net identification

- `--ground-net NET` explicit; else **auto-detect**: exact `GND` (case-insensitive)
  first, then a `gnd`/`ground` glob, over `board.pads_by_net()`.
- If none found → message + skip. If several plausible (e.g. `AGND`, `DGND`) →
  warn and require `--ground-net`. (Multiple planes is a later enhancement.)

### 2. Existing-pour guard

- If `board.zones` contains any **fill-enabled** zone (`zone_fill_nets(board)`
  non-empty, or any zone on the target net/layer) → **warn and skip** the whole
  feature ("board already has a copper pour; not adding a ground plane"). Per the
  requirement. *(Refinement option: only conflict when a pour exists on the target
  net/layer; default to the conservative skip-on-any-pour.)*

### 3. Pour boundary

- `poly = geometry.outline_to_polygon(board.outline)` → `poly.buffer(-margin)`
  (margin default = the board clearance, else 0.5 mm).
- Take the **largest** region if the inset yields a `MultiPolygon`; error if empty
  (margin too large). Use the **exterior ring** for the `(pts …)` in v1; board
  **cutouts/holes** are a documented limitation (KiCad will still respect copper it
  fills around; true keepout holes are a later enhancement).
- One **zone node per target layer** (cleaner than a multi-layer zone — each gets
  its own `filled_polygon`). `--ground-plane-layer` default **back** (`B.Cu`);
  `front`/`both` supported (2-layer board).

### 4. Zone-node builder — `pcb.make_zone_node(...)` (new, beside make_segment/via)

```python
def make_zone_node(board, net, layer, pts, *, clearance, min_thickness,
                   thermal_gap, thermal_bridge, hatch=0.5) -> SList:
    """Build a (zone …) outline node (no filled_polygon — KiCad fills it)."""
    # (net …)(layer …)(uuid …)(hatch edge …)(connect_pads (clearance …))
    # (min_thickness …)(fill yes (thermal_gap …)(thermal_bridge_width …)
    #                              (island_removal_mode 0))
    # (polygon (pts (xy x y) …))
```

Fill/clearance/thermal values come from the rules (the GND net class clearance;
sensible KiCad defaults otherwise). `(net …)` via `_net_ref_node`.

### 5. Ground connectivity — add vias where a GND island doesn't reach the pour

KiCad's fill auto-connects (thermal relief) any GND pad/track/via **on the pour
layer within the pour**. The gap to close: GND copper that lives **only on the other
layer**. Per *connected GND component*:

1. **Union-find over GND copper** (pads + segments + vias of the ground net),
   reusing the report.py approach, but tracking, per component, **which layers it
   has copper on and where** (through-hole pads count as all layers; vias bridge
   layers).
2. A component **reaches the pour** if it has copper on the pour layer at a point
   **inside the pour polygon**.
3. For each component that does **not** reach the pour: pick a point on the
   component's *other-layer* copper that lies inside the pour polygon, and place a
   **GND via** there (other-layer → pour-layer) so the fill ties it in. Snap to the
   nearest grid node and search outward for a spot where `grid.can_via(col, row,
   gnd_id)` holds (DRC-safe). If no free via spot is found near the island → **warn**
   that the net component couldn't be auto-connected (suggest manual), don't fail
   the run.

*(Most boards with through-hole GND pads already reach both layers, so this step
usually adds zero or few vias; it matters for all-SMD-on-front GND islands.)*

Existing routed GND **tracks are left in place** (valid GND copper; KiCad fills
around them). Ripping up now-redundant GND tracks is explicitly **out of scope**.

### 6. Stitching vias (optional, `--stitch-vias [PITCH]`)

- Lay a grid at `PITCH` (default ~5 mm) over the pour polygon; at each point place a
  GND via iff it's inside the pour **and** `grid.can_via(col,row,gnd_id)` (free of
  other-net copper/pads + clearance on all layers). Skip the rest.
- Most meaningful with `--ground-plane-layer both` (ties the two planes); for a
  single-layer pour it stitches GND copper to the plane at intervals. Off by default;
  document the EMI/return-path rationale.

### 7. CLI hook & refill (autoroute.py)

After routing finishes and before `write_board` (autoroute.py:696–711):

```python
gp_nodes, gp_warnings = (groundplane.build(board, grid, rules,
                            net=gnd_net, layers=gp_layers, margin=gp_margin,
                            stitch_pitch=stitch)              # [] if skipped/None
                         if args.ground_plane else ([], []))
for w in gp_warnings: rep.log(w); print(f"  ⚠ {w}")
nodes = _results_to_nodes(board, grid, final_results) + gp_nodes
pcb.write_board(board, out_path, new_nodes=nodes, strip_free_vias=True)

if fill_nets or args.ground_plane:        # force a refill when we added a pour
    pcb.try_refill_zones(out_path)        # (existing warn-if-no-kicad-cli path)
```

- The self-check (`clearance_violations`) runs as today; it validates all copper
  **except** the (KiCad-filled) pour — note this in the report so a "clean"
  self-check isn't over-read for the pour.
- Works with `--place` (post-route, independent) and `--cycles` (apply to the
  winning board before its write).

KiCad fill, which needs kicad-cli)
- `make_zone_node` round-trips: build → `write_board` → reload → `_parse_zones`
  yields the right net/layer/pts; node has no `filled_polygon`.
- Pour polygon = outline inset by margin (area shrinks; within the outline).
- Existing-pour guard: a board with a fill-enabled zone → returns no nodes + a

  pour → exactly one connecting via is added, DRC-safe (`grid.can_via`), inside the
  pour; a through-hole-GND board → zero connecting vias.
- Stitching: count/positions on a grid over the pour respect `can_via` and the pour
  boundary; off by default adds none.
- Post-add self-check (`clearance_violations`) stays clean (vias are DRC-safe);
  document that the pour itself isn't covered.

## Phasing

1. ✅ **Pour boundary + refill** — `--ground-plane` adds a single back-layer GND zone
   (outline inset by margin), existing-pour guard, GND auto-detect, force-refill.
2. ✅ **Connectivity vias** — guarantee every GND component reaches the pour layer.
3. ✅ **Stitching vias + multi-layer** — `--stitch-vias`, `--ground-plane-layer both/front`.

## Post-ship fixes

- **`thermal_bridge_width < min_thickness`**: KiCad silently dropped thermal relief
  spokes narrower than `min_thickness`, leaving THT pads unconnected. Fixed: clamp
  `thermal_bridge_width` to `max(clearance, min_thickness)`.
- **Connectivity via shorting B.Cu traces**: vias were placed at SMD pad centres
  without checking for other-net copper already routed on the pour layer. Fixed:
  build a B.Cu obstacle index (including freshly-routed `new_nodes` not yet in the
  board object) and reject conflicting positions; spiral-search outward for an
  alternative when the pad centre conflicts.
- **Component-level layer check**: the union-find used a per-position check instead
  of per-component, incorrectly flagging components containing THT pads as needing
  connectivity vias. Fixed: aggregate layers across all positions in each component.

## Risks & open questions

- **Pour clearance is KiCad's job.** PyAutoRoute's self-check can't see the fill
  (`board_obstacles` skips filled zones), so a clean self-check doesn't certify the
  pour. Mitigation: rely on `kicad-cli` refill/DRC; without it, ship an *unfilled*
  zone and warn. *(A future enhancement could teach `board_obstacles` to treat the
  zone outline as approximate copper for a rough self-check — but KiCad's fill is the
  source of truth.)*
- **Outline complexity.** `buffer(-margin)` on concave outlines / boards with
  cutouts can yield a `MultiPolygon` or empty geometry. v1: largest exterior region,
  error on empty; holes/keepouts deferred.
- **GND detection ambiguity** (`AGND`/`DGND`/multiple grounds). Default `GND`, flag
  to override, warn on ambiguity. Multiple simultaneous planes: later.
- **Via placement may fail** in a dense corner (no free via spot near an isolated
  GND island). Warn per-net rather than fail; suggest manual fix.
- **Redundant GND tracks.** Left in place (safe); auto-ripup is out of scope.
- **Both-layer pours** emit two zone nodes; confirm KiCad is happy with two
  single-layer GND zones over the same area (it is — standard practice with
  stitching).
- **Numbered-net boards.** `_net_ref_node` / the zone `(net …)` must use the right
  net code for older KiCad numbered-net files; covered by the existing net-style
  handling but worth a test on a numbered-net board.
- **GUI exposure** (a checkbox + net/layer/margin) is a natural follow-up; this plan
  is CLI-first.
